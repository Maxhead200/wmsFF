# Архитектура WMS LOGOFF

## Контур

WMS является главным источником складских остатков. 1С, маркетплейсы и файлы Excel считаются внешними источниками/получателями данных, но не управляют фактическим складским остатком напрямую.

## Backend

- NestJS API публикует REST `/api/v1`.
- PostgreSQL хранит справочники, складские операции, ledger и биллинг.
- Redis/BullMQ используется для фоновых задач: печать, импорты, синхронизация ТСД, интеграции.
- Prisma фиксирует схему данных и миграции.

## Stock ledger

Остаток не меняется "перезаписью количества". Каждая складская операция создаёт движение в `stock_movements`, а актуальный остаток считается по ledger и материализуется в `stock_balances`.

Так проще проверять расхождения, восстанавливать историю и безопасно синхронизировать offline-операции ТСД.

## Android-ТСД

Первый релиз работает online. Native Kotlin app использует generic keyboard wedge: сканер вводит штрихкод как клавиатура. В коде сразу есть outbox-модель, чтобы следующий этап добавил offline без переделки API.

## Безопасность

Реальные пароли, токены, API-ключи и production-дампы не попадают в репозиторий. Для клиентов все запросы фильтруются по `client_id`, а внутренние адреса хранения не отдаются в клиентский кабинет.
